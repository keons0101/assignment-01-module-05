# Nonrepudiation essay
**Harold Minotta**

The term nonrepudiation is used to describe the concept of making sure that a person or system can't deny that something really happened. In the context of technology, nonrepudiation helps to solve the issue of trusting something because it creates a digital record that can be verified later.

For example, in an online bookstore, users are constantly performing actions like adding books, sending forms, or making changes to the data. If someone says that they did not make any changes to the data or did not send any forms, then it becomes difficult to verify the truth without any logs or evidence. Keeping logs of events helps the developers to get an idea of what really occurred, making it easier to add nonrepudiation.

Basic logging is just the first step in nonrepudiation. There are many technologies that can be used to add nonrepudiation to an application, making it more difficult to modify the logs. Digital signatures can be used to verify that an individual sent a message, while hashing can be used to verify that the data was not altered. Security audits review activity over time, and some modern systems use blockchain or similar immutable records to keep a permanent and verifiable history of actions.
